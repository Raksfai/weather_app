# Atmos Weather App

[English](README.md) | Українська

Atmos — це погодний дашборд на Django, який показує поточну погоду, практичні поради щодо одягу, 5-денний прогноз, інтерактивні графіки, анімовані погодні фони, обране, порівняння міст і багатомовний інтерфейс.

Застосунок використовує OpenWeatherMap для погодних даних і зберігає легкий користувацький стан, наприклад обрані міста та міста для порівняння, у браузерному `localStorage`.

## Можливості

- Пошук поточної погоди за назвою міста.
- Відображення температури, відчутної температури, вологості, тиску, країни, опису та погодного емодзі.
- Поради щодо одягу на основі відчутної температури, вологості та погодних умов.
- 5-денний прогноз від OpenWeatherMap.
- Інтерактивний графік Chart.js для температури та вологості.
- Перемикання графіка й порад щодо одягу натисканням на картки днів прогнозу.
- Зміна анімованого фону сторінки відповідно до погоди вибраного дня.
- Збереження обраних міст у `localStorage`.
- Порівняння до 5 міст поруч.
- Порівняння поточної температури, вологості та тиску в окремих мініграфіках.
- Сортування таблиці порівняння за містом, температурою, вологістю або тиском.
- Багатомовний інтерфейс: англійська, українська, російська, німецька, іспанська та польська.
- Запит локалізованих описів погоди з OpenWeatherMap відповідно до активної мови інтерфейсу.

## Технології

- Python 3.12
- Django 6
- SQLite для локальної розробки
- OpenWeatherMap API
- Chart.js через CDN
- WhiteNoise для роздачі статичних файлів у production
- `uv` для залежностей і запуску команд
- Ruff для лінтингу

## Структура проєкту

```text
core/
  settings.py          Налаштування Django, i18n, статичні файли, middleware
  urls.py              Коренева маршрутизація
weather/
  views.py             Пошук погоди, обробка прогнозу, JSON endpoint порівняння
  urls.py              Маршрути застосунку
  tests.py             Django-тести з моками погодного API
  templates/weather/
    index.html         Головна сторінка, модальне вікно, перекладений UI
  static/weather/
    style.css          Layout, погодні теми, анімації, стилі модального вікна
    weather-chart.js   Основний графік прогнозу
    compare.js         compare localStorage, модальне вікно, графіки, таблиця
    favorites.js       Dropdown обраного через localStorage
locale/
  */LC_MESSAGES/       Джерела перекладів і скомпільовані файли повідомлень
```

## Змінні середовища

Створіть файл `.env` у корені проєкту.

```env
SECRET_KEY=your-django-secret-key
WEATHER_API_KEY=your-openweathermap-api-key
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
```

Обов'язкові змінні:

- `SECRET_KEY`: секретний ключ Django.
- `WEATHER_API_KEY`: API-ключ OpenWeatherMap.

Рекомендовані змінні:

- `DEBUG`: `True` для локальної розробки та `False` для production.
- `ALLOWED_HOSTS`: список дозволених хостів через кому для розгорнутих середовищ.

## Встановлення

Встановіть залежності з lock-файла:

```bash
uv sync
```

Застосуйте міграції:

```bash
uv run python manage.py migrate
```

Після змін у `.po` файлах скомпілюйте переклади:

```bash
uv run python manage.py compilemessages
```

## Локальний запуск

Запустіть Django development server:

```bash
uv run python manage.py runserver
```

Якщо у вашому середовищі за замовчуванням `DEBUG=False`, запускайте локально так:

```bash
DEBUG=True uv run python manage.py runserver 127.0.0.1:8000
```

Потім відкрийте:

```text
http://127.0.0.1:8000/
```

## Використання

1. Введіть назву міста в поле пошуку.
2. Перегляньте поточну погоду та 5-денний прогноз.
3. Використовуйте графік для перегляду температури й вологості.
4. Натискайте картки прогнозу, щоб перемикати дані графіка, поради щодо одягу та фонову тему.
5. Натисніть `⭐ Додати в обране`, щоб зберегти місто локально.
6. Натисніть `⚖️ Додати до порівняння`, щоб додати місто до списку порівняння.
7. Додайте щонайменше два міста, а потім натисніть `⚖️ Порівняти (N)`, щоб відкрити модальне вікно порівняння.
8. Використовуйте перемикач мови в хедері, щоб змінити мову інтерфейсу.

## Поради щодо одягу

Панель порад показує компактні рекомендації щодо одягу для поточного вибраного погодного дня.

Вхідні дані для рекомендацій:

- Відчутна температура для комфортних температурних діапазонів.
- Поточна або прогнозована вологість.
- OpenWeatherMap condition ID для дощу, снігу, грози, туману та ясних/хмарних варіантів.

Під час першого завантаження панель базується на поточній погоді. Коли вибрано картку прогнозу, панель оновлюється для цього дня прогнозу, використовуючи ту саму репрезентативну точку, що й картка прогнозу, зазвичай `12:00`, якщо вона доступна.

## Погодні фони

Тема сторінки базується на OpenWeatherMap condition IDs:

- Clear: сонячний анімований фон.
- Clouds: фон із хмарами, що повільно рухаються.
- Rain: падаючі краплі дощу.
- Snow: падаючий сніг.
- Thunderstorm: грозові хмари, дощ і блискавка.
- Mist та схожі туманні умови: приглушений туманний фон.

Коли вибрано день прогнозу, фон перемикається на погодну тему цього дня.

## Порівняння міст

Міста для порівняння зберігаються в браузерному `localStorage` під ключем `compare`.

Поведінка:

- Максимум 5 міст.
- Повторне натискання кнопки порівняння додає або прибирає поточне місто.
- Модальне вікно порівняння запитує свіжу поточну погоду з `/compare/?cities=...`.
- Міста з помилками показуються окремо, а успішні міста все одно відображаються.
- Три окремі графіки порівнюють температуру, вологість і тиск.
- Таблицю можна сортувати за містом, температурою, вологістю та тиском.

## Обране

Обрані міста зберігаються в браузерному `localStorage` під ключем `favorites`.

Dropdown обраного веде назад до збережених пошуків міст.

## Інтернаціоналізація

Проєкт використовує i18n на основі Django gettext.

Підтримувані мови:

- `en`: English
- `uk`: Ukrainian
- `ru`: Russian
- `de`: German
- `es`: Spanish
- `pl`: Polish

Файли перекладів розташовані тут:

```text
locale/<language>/LC_MESSAGES/django.po
locale/<language>/LC_MESSAGES/django.mo
```

Після редагування перекладів виконайте:

```bash
uv run python manage.py compilemessages
```

Активна мова також передається до OpenWeatherMap як параметр `lang`, щоб описи погоди були локалізовані, коли API це підтримує.

JavaScript-рядки інтерфейсу перекладаються через серверний JSON-словник у головному шаблоні.

## Тестування

Запустіть Django test suite:

```bash
uv run python manage.py test
```

Запустіть lint-перевірки:

```bash
uv run ruff check .
```

Поточні тести покривають:

- Успішний рендеринг сторінки погоди.
- Поточну погоду та дані графіка прогнозу.
- Групування прогнозу за датою.
- Поради щодо одягу для поточної погоди та вибраних днів прогнозу.
- Стан помилки для невідомого міста.
- JSON endpoint порівняння міст.
- Дедуплікацію порівняння та обмеження в 5 міст.
- Часткові помилки порівняння.
- Рендеринг i18n і перекладені JSON-помилки.
- Використання параметра OpenWeatherMap `lang`.

## Статичні файли

Для production-збору статичних файлів:

```bash
uv run python manage.py collectstatic
```

Коли `DEBUG=False`, WhiteNoise використовує `CompressedManifestStaticFilesStorage`, тому production-деплої мають виконувати `collectstatic`.

## Нотатки

- Не комітьте реальні секрети або API-ключі.
- Зовнішні погодні запити потрібно мокати в тестах.
- Стан браузера для обраного та порівняння є локальним для браузера користувача.
- Chart.js завантажується з CDN, тому для графіків потрібен доступ браузера до цього CDN.
